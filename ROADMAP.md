# Open-Unlearning 项目路线图

本文档概述了 Open-Unlearning 框架的发展方向和计划。

---

## 当前状态概览

| 类别 | 状态 |
|------|------|
| 基准测试 | 3 个主要基准 (TOFU, MUSE, WMDP) |
| 遗忘方法 | 14+ 种方法实现 |
| 评估指标 | 10+ 种指标 |
| 模型支持 | 14+ 种架构 |
| 测试覆盖 | 63 个单元测试 |
| 文档 | 完善的用户指南和贡献指南 |

---

## 路线图

### 阶段 1: 稳定性与质量 🔧

**目标**: 修复已知问题，提升代码质量

| 任务 | 优先级 | 状态 |
|------|--------|------|
| 修复 StepLog 循环缓冲区边界条件 bug | 高 | ✅ 已完成 |
| 修复 `_flatten()` 空列表边界情况 | 高 | ✅ 已完成 |
| 增加集成测试覆盖 | 中 | 待开始 |
| 添加 GPU 功能专项测试 | 中 | 待开始 |
| 实现基于属性的测试 (Hypothesis) | 低 | 待开始 |

### 阶段 2: 功能增强 🚀

**目标**: 扩展核心功能，提升易用性

#### 2.1 参数高效微调支持

| 任务 | 描述 |
|------|------|
| LoRA 集成 | 支持 LoRA 适配器进行高效遗忘 |
| QLoRA 支持 | 量化 + LoRA 以支持更大模型 |
| 适配器组合 | 多适配器融合与切换机制 |

#### 2.2 高级遗忘功能

| 任务 | 描述 |
|------|------|
| 低秩近似实现 | LMCleaner 中的低秩 Hessian 近似 |
| 在线/流式遗忘 | 支持增量式数据遗忘 |
| 参数回滚方法 | 基于 TIM 的参数状态回滚 |
| 多任务遗忘 | 同时遗忘多类数据的优化策略 |

#### 2.3 评估增强

| 任务 | 描述 |
|------|------|
| 扩展隐私攻击评估 | 更多 MIA 变体和对抗性测试 |
| 效率基准测试 | 标准化的计算开销对比 |
| 遗忘-效用权衡分析 | 自动化的帕累托前沿分析 |

### 阶段 3: 可扩展性 📈

**目标**: 支持更大规模的模型和分布式场景

| 任务 | 描述 |
|------|------|
| 70B+ 模型支持 | 优化内存管理以支持超大模型 |
| 分布式评估 | 跨节点并行评估基准测试 |
| 模型分片 | FSDP/DeepSpeed ZeRO-3 完整支持 |
| 批量实验管理 | 超参数搜索和实验编排 |

### 阶段 4: 生态系统建设 🌐

**目标**: 构建活跃的社区和丰富的资源

| 任务 | 描述 |
|------|------|
| 交互式排行榜 | 动态更新的方法对比平台 |
| 模型库扩展 | HuggingFace Hub 预训练模型增加 |
| 教程系列 | 视频教程和 Jupyter notebooks |
| API 服务 | 遗忘即服务 (Unlearning-as-a-Service) 原型 |

## 贡献指南

我们欢迎以下方向的贡献：

1. **新遗忘方法**: 使用 `community/methods/template/` 模板
2. **新基准测试**: 使用 `community/benchmarks/template/` 模板
3. **Bug 修复**: 查看 GitHub Issues
4. **文档改进**: 特别是中文文档

详细贡献流程请参考 [docs/contributing.md](docs/contributing.md)
